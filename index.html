HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • HTML elements are represented by tags.
  • HTML elements can have attributes that provide additional information about the element.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • When we want to center an element, we can use the margin property and set the left and right margins to auto.
  • The period preceding card designates this as a class selector. Any HTML element assigned to the class, card, will gain these property assignments from the CSS.
  • Class attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add .: adds all files to the staging area
  • git commit -m "message": commits the files to the local repository
  • git push origin branch-name: pushes the files to the remote repository
  • git pull origin branch-name: pulls the files from the remote repository
  • git checkout branch-name: switches to the specified branch
  • git branch: lists all branches
  • git branch -d branch-name: deletes the specified branch

JavaScript

  • each line of JavaScript code ends with a semicolon.
  • JavaScript is case sensitive.
  • In JavaScript, code runs from the first statement at the top of the page, down to the last statement at the bottom of the page
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Functions are reusable blocks of code that perform a specific task.
  • Parameters are variables that are used to store the data that's passed into a function.
  • Arguments are the actual data that's passed into a function.
